# Hermes Agent 記憶系統

## 概述

Hermes Agent 的記憶系統是其區別於其他 CLI Agent 的核心特徵之一。它允許 Agent 在會話之間保持對用戶偏好、項目環境和長期目標的記憶，無需每個新會話重新輸入背景資訊。

當你與 Hermes 開始新會話時，記憶內容會被自動注入系統提示，使 Agent 從第一句話就能說「我記得你上次在做的項目是...」。

## 記憶內容的類型

### 用戶 Profile（USER.md）

儲存用戶的基本信息與偏好：
- 姓名、語言、時區
- 溝通風格與偏好
- 設備與平台限制
- 職業背景

### 系統記憶（MEMORY.md）

儲存更動態的任務相關記憶：
- 當前項目的技術棧
- 最近的任務目標
- 環境配置與 known issues
- API keys 和訪問憑據位置

### 對話歷史壓縮

每次會話結束時，當前對話會被總結並可選擇性保存。長期歷史通過 session_search 工具可按關鍵詞檢索。

## 後端選項

Hermes 支持多種記憶後端：

| 後端 | 特點 | 適用場景 |
|------|------|----------|
| 內置（默認） | 基礎鍵值存儲，無需額外配置 | 一般用戶 |
| Honcho | 結構化用戶 Profile，更規範 | 需要嚴格用戶畫像 |
| Mem0 | 專業長期記憶系統，語義理解 | 高級用戶、研究用途 |
| 自定義 | 可接入任何兼容後端 | 開發者 |

## 後端切換

hermes memory setup
hermes memory status
hermes memory off

或在 config.yaml 中配置：

memory:
  provider: mem0
  memory_enabled: true
  user_profile_enabled: true

## 記憶工具操作

### 保存記憶

memory(action="add", target="memory", content="這是重要事實")
memory(action="add", target="user", content="用戶偏好某種方式")

### 更新記憶

memory(action="replace", target="memory",
       old_text="舊內容",
       content="新內容")

### 刪除記憶

memory(action="remove", target="memory", old_text="要刪除的內容")

### 檢索記憶

session_search(query="之前做過的項目")

## 已知限制：記憶工具的 matching 問題

重要：記憶工具內部並非純文本存儲，而是有結構化格式。memory remove 和 memory replace 需要精確匹配內部存儲格式，否則返回「No entry matched」。

實用解決方案：

1. 使用 replace 而非 remove（replace 要求 content 非空，更容易成功）
2. 直接編輯 MEMORY.md 文件（如果是磁盤後端）
3. 利用 session_search 確認實際存儲內容後再操作
4. 備份重要偏好到技能文件中（技能比記憶更可靠）

## 最佳實踐

### 應該記住的內容

適合放記憶的：
- 用戶偏好（如：不喜歡複雜方案、只用 iPhone）
- 環境事實（如：NAS 在 Z2 Pro 上、數據庫是 Neon PostgreSQL）
- 項目 context（如：這個項目用 Flask + SQLite）
- API 位置和鑰匙（如：Spotify Client ID 在 .env）

不適合放記憶的：
- 具體任務的 intermediate state（每次會話都會變）
- 已經在文檔/代碼中的內容
- 臨時調試信息
- 密碼（用 .env，別放記憶）

### 記憶膨脹問題

記憶會隨時間累積，可能變得過時或過大。定期清理：

1. 每季回顧：檢查 MEMORY.md 中是否有過時信息
2. 主動刪除：發現錯誤時立即修正
3. 結構化組織：使用清晰的 section 分隔

### 與技能的配合

記憶告訴 Agent 環境是什麼，技能告訴 Agent 遇到情況該怎麼做。兩者配合：

記憶：項目使用 Flask + SQLite
         ↓
技能：Use when debugging Flask applications with SQLite

## 檢視當前記憶

hermes memory status

或在會話中直接詢問：「我的記憶裡有什麼關於項目的信息？」

## 遷移與備份

更換設備或重裝時：
hermes profile export <name>
hermes profile import <file>

或直接備份內存文件：
cp ~/.hermes/profiles/default/memory.json ~/backup/
