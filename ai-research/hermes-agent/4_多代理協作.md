# Hermes Agent 多代理協作

## 概述

Hermes Agent 支援兩種多代理協作模式，適用於不同的場景需求。理解兩者的差異是有效使用多代理能力的關鍵。

## 兩種模式的核心對比

| 維度 | delegate_task | 獨立 Hermes 進程 |
|------|--------------|-----------------|
| 隔離性 | 獨立對話，共享父進程 | 完全獨立進程 |
| 持續時間 | 分鐘級 | 小時/天級 |
| 工具訪問 | 父進程工具子集 | 完整工具訪問 |
| 交互性 | 不可交互 | 可 PTY 交互 |
| 適用場景 | 快速並行子任務 | 長期自主任務 |

## delegate_task：輕量並行

### 基本語法

delegate_task(
    goal="具體任務描述",
    context="詳細背景信息，包括：任務目標、約束條件、期望輸出格式、驗證方法",
    toolsets=['web', 'terminal', 'file'],
    role="leaf"
)

### 何時使用

適合用 delegate_task 的場景：
- 需要同時研究多個方向（並行加速）
- 任務可完全拆解為獨立子任務
- 需要保持父會話 context 但要隔離子任務
- 批量處理多個相似項目

不適合用 delegate_task 的場景：
- 需要用戶即時反饋（subagent 無法問問題）
- 任務結果需要二次確認（不能假設 subagent 說完成了就是真的完成）

### 關鍵原則：完整上下文

Subagent 沒有父會話的完整 context，必須在 context 參數中提供一切必要信息。

不足：goal="寫一個 API"，context="用 Flask"

完整：goal="為 user management 服務編寫 REST API"，context="項目：~/projects/user-service，技術棧：Flask 2.x + SQLAlchemy + SQLite，現有用戶 model：User(id, email, password_hash, created_at)，要求：POST /users、GET /users/<id>、需要 JWT 認證，驗證：所有端點用 pytest 測試"

### 結果驗證

Subagent 的結果需要二次確認。調用後：
1. 檢查返回的 summary
2. 驗證實際輸出（如：文件是否存在、API 是否正常）
3. 不要假設「完成了」等於「正確完成了」

## 獨立進程：長期自主任務

### 透過 tmux 實現

Hermes 使用 prompt_toolkit，需要真實終端。使用 tmux 提供 PTY：

tmux new-session -d -s agent1 -x 120 -y 40 'hermes'
sleep 8
tmux send-keys -t agent1 'Build REST API' Enter
tmux capture-pane -t agent1 -p
tmux kill-session -t agent1

### Worktree 模式：並行開發同一 Repo

hermes -w 開啟 git worktree，隔離的 git 環境，避免同時操作同一 Repo 產生 conflicts。

tmux new-session -d -s backend 'hermes -w'
tmux new-session -d -s frontend 'hermes -w'

## 雙階段審核模式

對於高質量要求的任務，推薦兩階段審核：

任務拆分
    ↓
Step 1: 實作 subagent（implementer）
    ↓
Step 2: Spec compliance reviewer
    ↓ 失敗 -> 返回 Step 1 修復
    ↓
Step 3: Code quality reviewer
    ↓ 失敗 -> 修復 -> 重新審核
    ↓
完成

## 常見陷阱

1. context 不完整：導致 subagent 做出錯誤假設
2. 假設 subagent 完成等於正確完成：必須驗證
3. 過度細分：每個小任務都用 subagent，overhead 太大
4. 順序依賴卻並行：有依賴關係的任務不應並行
5. 父會話依賴子會話 state：subagent 結束後 state 消失

## 實用命令速查

delegate_task（代碼中）：
delegate_task(goal="...", context="...", toolsets=[...])

tmux 獨立會話：
tmux new-session -d -s <name> 'hermes'
tmux send-keys -t <name> '<command>' Enter
tmux capture-pane -t <name> -p
tmux kill-session -t <name>

Worktree 模式：hermes -w
