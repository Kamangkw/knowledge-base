# Hermes Agent 技能系統

## 概述

Hermes Agent 的技能系統是其最具差異化的設計。傳統 AI Agent 每個會話都是全新的，無法從過去的經驗中學習。而技能允許你將解決問題的完整工作流程保存為一個持久化的文件，讓未來任何會話都能直接加載使用，無需重新解釋背景。

**技能與記憶的核心區別：**

| 維度 | 記憶（Memory） | 技能（Skills） |
|------|---------------|----------------|
| 儲存內容 | 事實、偏好、配置 | 工作流程、操作步驟 |
| 顆粒度 | 原子化 | 結構化 |
| 觸發方式 | 被動讀取 | 主動加載 |
| 適用場景 | 長期偏好、項目背景 | 複雜任務流程 |

兩者互補：記憶告訴 Agent「你是誰」，技能告訴 Agent「遇到 X 情況時該怎麼做」。

## SKILL.md 結構詳解

每個技能都是一個獨立的 SKILL.md 文件，包含 YAML frontmatter 和 Markdown 正文。

### 必須欄位

- **name**：lowercase + hyphens，≤64 字元
- **description**：必須以「Use when ...」開頭，精確描述觸發情境，≤1024 字元
- version、author、license、metadata（建議填寫）

### 描述寫作黃金法則

description 是技能系統的核心——它決定了何時觸發這個技能。必須以「Use when ...」開頭，精確描述觸發情境，而非具體任務。

```
正確：Use when debugging memory issues in Hermes Agent or improving agent performance.
錯誤：Debug memory issues.              （太抽象，不知何時用）
錯誤：Fix a bug in the agent.          （太寬泛，隨時都能扯上關係）
```

### 推薦的 SKILL.md 結構

```
# Title

## Overview
一兩段話：這個技能是什麼、為何需要它。

## When to Use
- Bulleted 觸發情境
- "Don't use for:" 反向觸發

## 具體內容區塊
- Quick-reference tables
- Exact commands with code blocks
- Hermes-specific recipes

## 常見陷阱
編號列表：常見錯誤與修復方法

## 驗證清單
- [ ] Checkbox 列表
```

## 技能分類與應用場景

技能可分為五大類型：

### 1. 工具型

封裝特定工具的使用方法。

```
觸發：Use when working with GitHub API or managing pull requests.
範例：GitHub PR Workflow、Spotify API Integration
```

### 2. 知識型

儲存特定領域的深入知識。

```
觸發：Use when researching or explaining <topic>.
範例：ClickHouse SQL Patterns、Macau Leave System
```

### 3. 工作流型

定義複雜任務的標準執行步驟。

```
觸發：Use when implementing a new feature from plan to deployment.
範例：Subagent-Driven Development、TDD Workflow
```

### 4. 整合型

連接外部系統或平台的標準流程。

```
觸發：Use when setting up or debugging Home Assistant integrations.
範例：HA Apple TV Control、LG webOS TV
```

### 5. 對話型

指導特定類型對話的處理方式。

```
觸發：Use when <user> shares <type of content> or asks about <topic>.
範例：Tacit Knowledge Learning、Active Caring
```

## 技能生命週期

### Phase 1：創建（Create）

完成一個複雜任務後，主動問：「要不要存成技能？」

創建位置有兩種：
- 用戶本地：`~/.hermes/skills/<category>/<name>/SKILL.md`
- Repo 共享（需要 commit 到代碼庫）

### Phase 2：測試（Test）

在新會話中加載並驗證：

```
/skill my-skill-name
# 或
hermes -s my-skill-name
```

驗證清單：
- [ ] 觸發條件準確
- [ ] 步驟完整
- [ ] 命令準確（實際運行過，不是估計）
- [ ] 錯誤處理已定義

### Phase 3：發布（Publish）

對於有價值的技能，可以發布到 Skills Hub：

```bash
hermes skills publish <path>
```

其他人可以通過以下方式安裝：
```bash
hermes skills install <skill-id>
hermes skills install https://github.com/user/repo/blob/main/SKILL.md
```

### Phase 4：維護（Maintain）

技能需要隨環境變化而更新。常見維護觸發：
- 外部 API 變更
- 新的最佳實踐
- 用戶反饋指出不足

## 高級模式

### 技能鏈（Skill Chaining）

多個技能可以串聯使用，一個的輸出引導下一個：

```
/skill research      # 做初步 research
/skill synthesize    # 綜合研究結果
/skill write         # 寫成文檔
```

### 條件式加載

在複雜會話中，可以根據任務類型動態加載技能。

### 技能組合

一個大的工作流可以拆分為多個協作的技能：

```
subagent-driven-development
    ├── writing-plans
    ├── requesting-code-review
    └── test-driven-development
```

## 常見陷阱

1. **描述太抽象**：無法判斷何時該用這個技能
2. **Scope 太大**：一個技能塞進太多功能，違背單一職責
3. **依賴外部狀態**：技能內假設了某些 context 未被明確說明
4. **從未測試**：寫完就放著，不知道實際能不能用
5. **重複現有技能**：在創建前未檢查是否已有類似的

## 實用命令

```bash
hermes skills list              # 列出所有已安裝技能
hermes skills search <query>   # 搜尋 Skills Hub
hermes skills install <id>     # 安裝技能
hermes skills check            # 檢查更新
hermes skills update           # 更新過時的技能
```
