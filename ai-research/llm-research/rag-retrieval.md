# RAG檢索增強生成：向量檢索、知識庫構建與最佳實踐

## 概述

檢索增強生成（Retrieval-Augmented Generation，簡稱 RAG）是近年來大語言模型（LLM）領域最重要的技術突破之一。傳統的 LLM 面臨一個根本性的局限：知識被「凝固」在訓練數據中，無法即時更新，且容易產生幻覺（hallucination）——即生成看似流暢但實際上並不準確的內容。RAG 通過將 LLM 與外部知識檢索系統相結合，有效地解決了這一問題。

RAG 的核心思想非常直觀：在回答用戶問題時，首先從一個外部知識庫中檢索與問題相關的文檔，然後將這些文檔作為上下文提供給 LLM，輔助其生成答案。這種「先檢索、後生成」的模式，使得 LLM 能夠訪問動態更新的知識，同時也提供了答案的來源依據，大幅提升了可解釋性和可信度。

## 向量檢索基礎

### 詞嵌入（Word Embeddings）

理解 RAG 的第一步是理解向量表示（vector representation）。在自然語言處理中，「詞嵌入」是將文字轉化為稠密向量（dense vector）的技術，使得語義相近的詞在向量空間中彼此接近。

Word2Vec（2013）開創了詞嵌入的先河，通過神經網絡學習詞的分布式表示。Word2Vec 的兩大算法——Skip-gram 和 CBOW——至今仍是理解詞嵌入的重要基礎。然而，Word2Vec 的一個根本局限在於它無法處理一詞多義（polysemy）的問題。

BERT（2018）的出現標誌著上下文感知詞嵌入（contextualized embeddings）的時代。與 Word2Vec 的靜態詞嵌入不同，BERT 生成的詞向量會根據詞所處的上下文動態變化。例如，「bank」在「river bank」和「savings bank」中的 BERT 表示會有所不同，這使得語義表示更加精確。

### 句子與文檔嵌入

現代 RAG 系統通常使用句子級或文檔級的嵌入模型，而非僅僅依賴詞嵌入。這些模型包括：

**Sentence-BERT (SBERT)**：專為句子匹配任務訓練的 BERT 變體，能生成语义相似的句子在向量空间中彼此接近的表示。SBERT 的出現使得大規模語義相似度搜索成為可能。

**Instructor**：一個指令感知的句子嵌入模型，能根據給定的任務指令（如「為信息檢索任務編碼這個句子」）生成定制化的嵌入，在多種檢索基準上達到了最先進的性能。

**OpenAI 的 text-embedding-ada-002** 及最新的 **text-embedding-3** 系列：這些大規模訓練的嵌入模型在廣泛的語義文本相似度（STS）任務上表現優異，且經過優化適用於實際應用場景。

### 向量索引與搜索

將文檔轉化為向量後，下一個挑戰是如何在高維空間中快速找到與查詢向量最相似的文檔。常用的索引技術包括：

**Faiss (Facebook AI Similarity Search)**：Meta 開源的向量檢索庫，支援多種索引策略，包括 IVF（倒排文件索引）和 PQ（乘積量化）壓縮，能在數十億級別的向量集合中實現毫秒級檢索。

**HNSW (Hierarchical Navigable Small World)**：一種基於圖的近似最近鄰（ANN）算法，通過構建多層圖結構實現高效的最近鄰搜索。HNSW 是現在最流行的 ANN 索引之一，廣泛應用於各種向量數據庫中。

**SCaNN**：Google 開源的向量檢索庫，採用了激進的量化壓縮技術，在保持高精度的同時實現了極高的檢索速度。

## RAG 的核心組件

### 典型 RAG 流程

一個標準的 RAG 系統包含以下步驟：

1. **知識庫構建**：將原始文檔（PDF、網頁、資料庫等）進行分塊（chunking），然後使用嵌入模型將每個塊轉化為向量，存入向量數據庫。

2. **查詢處理**：當用戶提出問題時，使用相同的嵌入模型將問題轉化為向量。

3. **相似度檢索**：在向量數據庫中查找與問題向量最相似的 K 個文檔塊。

4. **上下文增強**：將檢索到的文檔塊與原始問題組合成 prompt，提供給 LLM。

5. **答案生成**：LLM 根據增強後的 prompt 生成最終答案。

### 文檔分塊策略

文檔分塊（chunking）是 RAG 系統設計中的關鍵環節。分塊太小可能丢失上下文，分塊太大則可能引入過多噪聲。

**固定大小分塊**：最簡單的方法，設定每個塊的 token 數量（如 512 tokens）。優點是簡單可控，缺點是可能切斷語義單元。

**語義分塊**：使用 NLP 技術識別文檔中的語義單元（如段落、句子），在語義邊界處進行分割。這種方法能更好地保留上下文完整性。

**層次分塊**：構建多粒度的塊結構——每個段落是一個塊，相關段落又組成更大的塊。檢索時可以根據需要選擇不同粒度的上下文。

## 高級 RAG 技術

### HyDE (Hypothetical Document Embeddings)

HyDE 是一種巧妙的檢索策略。其核心思想是：首先讓 LLM 根據問題生成一個「假設性答案」文檔，然後使用這個假設性文檔而非原始問題去檢索相關知識。HyDE 的原理在於：假設性答案與真實知識文檔在語義空間中可能比原始問題更加接近。

這種方法的直覺是：用戶的問題可能與知識庫中的表述方式不同，但問題想要的答案與知識庫中的內容在語義上是相關的。讓 LLM 先生成一個「可能是答案」的文檔，可以捕捉到這種語義相關性。

### Self-RAG

Self-RAG（Schrapper et al., 2023）是 RAG 領域的另一重要進展。傳統 RAG 不區分檢索結果的質量，一律將檢索到的文檔用於生成。Self-RAG 引入了一個「反思標記」（reflection token）機制，讓 LLM 自己判斷：
- 是否需要檢索
- 檢索到的文檔是否相關
- 生成內容是否被檢索結果所支持

這種自我審查機制有效減少了無關檢索帶來的噪聲，並提高了答案的事實準確性。

### 重排序（Reranking）

初始檢索後，返回的文檔可能包含大量噪聲或不那麼相關的內容。重排序階段使用一個更大的、更精確的模型（如 cross-encoder）對初始結果進行重新排序，將最相關的文檔排在最前面。

常用的重排序模型包括：
- **Cohere Rerank**：專門為搜索重排序訓練的商業模型
- **BGE Reranker**：智源開源的中文重排序模型
- **DUPE Reranker**：基於複製檢測的簡單但有效的方法

## 多模態 RAG

隨著多模態大模型的發展，多模態 RAG 也成為一個活躍的研究方向。多模態 RAG 能夠處理不僅包含文字，還包含圖像、表格、影片等內容的知識庫。

典型的方法是將不同模態的內容分別轉化為向量表示（圖像使用 CLIP 等 vision-language 模型），然後在多模態向量空間中進行檢索。更多高級的方法則探索跨模態的語義對齊和推理。

## RAG 評估

RAG 系統的評估通常從以下幾個維度進行：

**答案精確度（Answer Accuracy）**：答案是否與事實相符？
**答案相關性（Answer Relevance）**：答案是否針對用戶問題？
**上下文利用率（Context Utilization）**：檢索到的文檔是否被有效利用？
**檢索質量（Retrieval Quality）**：檢索到的文檔是否與問題相關？

RAGAS（Es et al., 2023）是專門為 RAG 評估設計的框架，提供了基於 LLM 的自動評估方法。此外，BERGEN、ARES 等評估框架也各有特色。

## RAG 系統的常見陷阱

### 分塊策略不當

分塊太大會引入過多噪聲，分塊太小則丢失上下文。一個常見的問題是分塊沒有考慮到語義邊界——例如在一個論證的中間切斷，導致 LLM 只能看到論證的一半。

### 檢索質量問題

向量相似度並不等同於語義相關性。一個查詢可能與某個文檔在向量空間中很接近，但實際上並不是回答該問題所需的信息。

### 上下文窗口限制

當檢索到大量文檔時，可能超出 LLM 的上下文窗口限制。這時需要精心地選擇和過濾文檔，而非簡單地將所有檢索結果塞入 prompt。

### 領域適配不足

通用嵌入模型在特定領域（如醫療、法律、金融）可能表現不佳。這時需要考慮使用領域特定的嵌入模型，或者對通用模型進行微調。

## RAG 的未來發展

RAG 技術正在快速演進。幾個值得關注的方向包括：

**更深層次的檢索-生成融合**：不只是簡單地將檢索結果附加到 prompt，而是讓 LLM 在生成過程中動態地查詢和利用外部知識。

**結構化知識的融合**：將傳統的知識圖譜、知識庫與向量檢索相結合，發揮各自優勢。

**自主學習和持續更新**：讓 RAG 系統能夠從用戶反饋中持續學習和更新知識庫。

## 結論

RAG 代表了一種重要的 AI 系統設計範式——將大語言模型的強大生成能力與外部知識的動態訪問能力相結合。通過合理的系統設計和持續的優化，RAG 能夠顯著提升 LLM 在知識密集型任務中的表現，同時提高答案的可解釋性和可信度。

隨著嵌入模型、向量數據庫和 LLM 技術的不斷進步，RAG 系統的性能和易用性還將持續提升，使其成為企業級 AI 應用的標準組件。
