# 設計思維理論基礎與演化歷程

## 基本資訊

| 項目 | 內容 |
|------|------|
| 類型 | 創新方法論 / 設計理論 |
| 起源 | 1960年代設計方法論運動 |
| 主流化 | IDEO創辦人David Kelley於1991年創立IDEO，2005年史丹福d.school成立 |
| 核心主張 | 以人為本的創新，結合同理心、實驗與迭代 |

## 核心概念

### 1. 以人為本（Human-Centered）

設計思維的核心假設：所有設計決策最終都應服務人類需求。

- **觀察」重於「問卷」**：設計師需要沉浸式觀察用戶行為，而非依賴自我報告
- **潜在需求」比「表達需求」更重要**：用戶往往不知道自己要什麼（亨利·福特：「如果問顧客他們想要什麼，他們會說更快的馬」）
- **情感設計」與「功能設計」並重**：產品需要同時滿足理性與情感需求

### 2. 雙鑽石模型（Double Diamond）

英國設計委員會（British Design Council）2004年提出的四階段框架：

```
第一個鑽石：擴散 → 聚焦
    發現（Discover）→ 定義（Define）

第二個鑽石：擴散 → 聚焦
    發展（Develop）→ 交付（Deliver）
```

- **發現階段**：進行用戶研究、市場調查、趨勢分析，擴大問題理解範圍
- **定義階段**：綜合洞察，確定要解決的正確問題（right problem）
- **發展階段**：頭腦風暴、快速原型、多種解決方案
- **交付階段**：測試、迭代、最終方案

### 3. 設計思維三空間（Three Spaces）

| 空間 | 描述 | 關鍵活動 |
|------|------|----------|
| **靈感空間（Inspiration）** | 理解人與環境 | 田野調查、觀察、訪談 |
| **構思空間（Ideation）** | 生成可能性 | 頭腦風暴、原型、實驗 |
| **實現空間（Implementation）** | 推向世界 | 商業模式、推廣、迭代 |

### 4. 設計思維與傳統問題解決的對比

| 維度 | 傳統線性方法 | 設計思維 |
|------|-------------|----------|
| **問題** | 預先定義 | 持續探索與重新定義 |
| **解決方案** | 單一最佳解 | 多樣化候選方案 |
| **失敗觀** | 避免失敗 | 快速失敗、快速學習 |
| **用户** | 被動接受者 | 積極共同創造者 |
| **迭代** | 最後階段才優化 | 贯穿全程 |

## 理論根源

### 1. 設計方法論運動（1960-70年代）

- **Herbert Simon**（1969）《人工科學》：設計是一種將現實轉向理想狀態的行動
- **Bryan Lawson**：設計師使用「反溯推理」（backwards reasoning），從解決方案回推問題
- **Bruce Archer**：系統化設計過程的重要性

### 2. 認知科學影響

- **Donald Schön**「反映式實踐者」（reflective practitioner）：專業知識來自行動中反思
- **Kelley**與**Brown**強調「專家魔術」（Expertise of Designers）：設計師的隱性知識

### 3. 設計研究（Design Research）

- **Herbert Simon**的「人工科學」：設計是創造人工物以實現目標的科學
- **Tillmann Juda**：設計思維是「對未知事物的系統化探索」

## 批評與局限

### 1. 過度簡化的風險

批評者認為設計思維被過度商品化：
- 從複雜的設計過程簡化為「同理心+原型+迭代」的口號
- 忽略深層的批判性設計思考

### 2. 組織文化的阻力

- 設計思維要求「心理安全感」——允許失敗的環境
- 官僚體制、短期績效壓力與設計思維格格不入

### 3. 「以人為本」的複雜性

- 「人」是誰？不同用戶群可能有衝突需求
- 設計師自身的偏見會影響研究與詮釋

### 4. 對創意生成的誤解

- 頭腦風暴被濫用，缺乏結構化引導
- 創意不僅是「自由聯想」，也需要紀律與深度

## 實際應用

### 企業創新

- **IDEO**：為蘋果設計第一款滑鼠、為飛利普設計醫療設備
- **SAP**：將設計思維融入軟件開發流程
- **Airbnb**：用設計思維重新定義旅遊住宿體驗

### 社會創新

- **貧民窟居民服務**： IDEO與非營利組織合作改善低收入社區
- **醫療創新**： 設計思維用於改善患者體驗、降低醫療錯誤
- **教育創新**： d.school的「教育設計」項目

### 公共服務

- **丹麥哥本哈根**： 用設計思維重塑公共服務
- **英國政府**： 行為洞察團隊（Behavioral Insights Team）結合設計思維

## 與其他創新方法的比較

| 方法 | 起源 | 核心 | 適用場景 |
|------|------|------|----------|
| **設計思維** | 設計 | 以人為本、迭代 | 不確定問題、需創新突破 |
| **精益創業** | 創業 | 快速驗證、最小可行產品 | 新產品開發、商業模式驗證 |
| **六西格瑪** | 工程 | 減少缺陷、流程優化 | 現有流程改進、質量控制 |
| **敏捷開發** | 軟件 | 迭代交付、靈活性 | 軟件開發、產品迭代 |
